OVERVIEW

왜 이동부터 만들었나

모든 RPG의 시작은 "캐릭터가 내 손끝에서 움직인다"는 감각이에요. MazeRPG의 첫 스테이지는 미로를 걸어다닐 캐릭터의 이동 · 시점 전환 시스템을 만드는 것이었어요. UE5의 최신 입력 방식인 Enhanced Input System을 기반으로 구현했어요.

목표 👉 WASD로 걷고, 마우스로 카메라를 돌리는 기본 3인칭 이동 완성하기.
SETUP

Enhanced Input 설정

// MazeCharacter.h
UPROPERTY(EditAnywhere, Category="Input")
UInputMappingContext* DefaultMappingContext;

UPROPERTY(EditAnywhere, Category="Input")
UInputAction* MoveAction;

UPROPERTY(EditAnywhere, Category="Input")
UInputAction* LookAction;
  • IMC (Input Mapping Context) — 키보드/마우스 입력을 "행동"으로 묶어주는 설정 묶음
  • IA_Move — WASD 입력값을 2D 벡터로 전달하는 액션
  • IA_Look — 마우스 이동값을 받아 카메라 회전에 사용
여백 메모 ✏️ 기존 Input(Axis Mapping) 대신 Enhanced Input을 쓴 이유는, 나중에 상황별로(예: 전투 중엔 다른 키 배치) 스위칭하기 훨씬 편해서예요.
IMPLEMENT

이동 로직, 이렇게 흘러가요

① 입력 감지 EnhancedInputComponent
->BindAction(MoveAction, ...)
② 값 수신 Value.Get<FVector2D>()
③ 방향 계산 GetControlRotation()
기준으로 Forward/Right 벡터 계산
④ 이동 적용 AddMovementInput
(Direction, Value) 
void AMazeCharacter::Move(const FInputActionValue& Value)
{
    const FVector2D MoveInput = Value.Get<FVector2D>();

    const FRotator ControlRot(0, Controller->GetControlRotation().Yaw, 0);
    const FVector Forward = FRotationMatrix(ControlRot).GetUnitAxis(EAxis::X);
    const FVector Right   = FRotationMatrix(ControlRot).GetUnitAxis(EAxis::Y);

    AddMovementInput(Forward, MoveInput.Y);
    AddMovementInput(Right,   MoveInput.X);
}
포인트 👉 카메라가 보는 방향 기준(Yaw)으로 이동 방향을 계산해야, 시점을 돌려도 "앞으로"가 항상 화면 위쪽이 돼요.
