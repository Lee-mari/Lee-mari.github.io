NAMING RULES

변수 이름, 아무렇게나 지으면 안 된다

변수는 이름표가 붙은 상자이지만, 그 이름표에는 지켜야 할 규칙이 있다. 크게 3가지를 지키지 않으면 컴파일 오류가 난다.

1
숫자로 시작할 수 없다

컴파일러가 코드를 올바르게 해석하기 위한 규칙이다.

2
키워드는 사용할 수 없다

C++이 이미 문법으로 쓰고 있는 단어는 변수명으로 못 쓴다.

3
공백(white space)을 사용할 수 없다

변수 이름 중간에 띄어쓰기가 들어가면 안 된다.

RULE 1

1. 숫자로 시작할 수 없다

언어의 구문 규칙에 따라 컴파일러가 코드를 올바르게 해석하고, 가독성을 유지하기 위해서이다.

🚫 잘못된 이름 
int 1age = 5;
// 오류: 숫자로 시작할 수 없음
⭐ 올바른 이름 
int age1 = 5;
// 숫자는 뒤에 와야 함
RULE 2

2. C++에서 사용하고 있는 키워드는 사용할 수 없다

이미 문법적인 의미로 예약되어 있는 단어(키워드)는 변수 이름으로 쓸 수 없다.

int
자료형
for
반복문
if
조건문
class
클래스
return
반환
주의 👉 int int = 5;처럼 키워드 자체를 변수명으로 쓰면 컴파일 오류가 난다.
RULE 3

3. white space(공백)를 사용할 수 없다

변수 이름 중간에 공백을 사용할 수 없다.

🚫 잘못된 변수 이름 (공백 포함) 
int my variable = 5;
// 오류: 변수 이름에 공백이
// 포함되어 있습니다.
⭐ 올바른 변수 이름 (공백 없음) 
int my_variable = 5;
// 밑줄로 단어를 구분

int myVariable = 10;
// 카멜 표기법 사용
정리하면 👉 단어를 여러 개 쓰고 싶을 땐 밑줄(_)로 잇거나, 두 번째 단어부터 대문자로 시작하는 카멜 표기법을 쓰면 된다.
