WHY

왜 필요한가?

김밥을 100줄 만다고 생각해보면, "한 줄 말기"를 100번 반복하는 거잖아요. 코드도 똑같은 동작을 여러 번 할 때, 매번 똑같이 써주는 대신 for문 하나로 "몇 번 반복할지"만 정해주면 돼요.

정리하면 👉 "같은 일을 N번 하고 싶을 때" 쓰는 도구예요.
여백 메모 ✏️ 사실 저도 처음엔 i가 왜 자꾸 나오는지 몰랐어요. "인덱스"의 i 래요.
STRUCTURE

기본 구조 뜯어보기

for (int i = 0; i < 5; i++)
{
    // 이 안의 코드가 5번 반복돼요
}
  • ① int i = 0 — 몇 번째인지 세는 번호표 만들기 (0부터 시작)
  • ② i < 5 — "5보다 작을 때까지만" 계속하라는 조건
  • ③ i++ — 한 바퀴 돌 때마다 번호표 +1
RUN IT

실제로 돌려보면?

3번 반복하도록 돌리면, 한 바퀴씩 이렇게 진행돼요 (옆으로 밀어서 보기 →)

i = 0몬스터 0번 소환!
i = 1몬스터 1번 소환!
i = 2몬스터 2번 소환!
i = 3 → 조건 거짓반복 종료 🛑
주의! 👉 0부터 시작해서 3번 도니까 0, 1, 2 까지만 나와요.
REAL USE

게임 개발에서는 이렇게 써요

"

스테이지 매니저가 몬스터 스포너 여러 개를 배치할 때, 하나하나 손으로 다 안 쓰고 이렇게 돌려요.

for (int i = 0; i < SpawnerArray.Num(); i++)
{
    SpawnerArray[i]->ActivateSpawner();
}
비유하면 👉 던전에 스포너가 10개든 100개든, "다 켜!"라고 한 번에 말하는 느낌이에요.
